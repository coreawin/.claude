## 확정된 결정사항

### 출력 형식
- 편집 가능한 `.pptx` 파일로 직접 생성 (결정일: 2026-03-19)

### 디자인 수준
- C (혼합): 보고서는 내용 중심, 제안서/피칭은 디자인 중심 (결정일: 2026-03-19)

### 외부 도구 후보
- **NotebookLM**: .pptx 직접 내보내기 가능(2026.02~), 인포그래픽 10가지 비주얼 프리셋 지원, Enterprise API(alpha) 및 비공식 Python API(notebooklm-py) 존재
- **BananaX (furoku.github.io)**: 300개+ 인포그래픽 AI 프롬프트 라이브러리, 평가기준 5항목(가독성/계층구조/일관성/분위기/테마적합성)

### 생성 워크플로우 (C - 혼합)
- 구조 슬라이드(텍스트/표/차트): python-pptx 직접 생성
- 인포그래픽 슬라이드: NotebookLM → 이미지 출력 → pptx 삽입 (편집 불가 이미지)
- BananaX 301개 프롬프트 로컬 저장 → 콘텐츠 유형별 자동 추천
- (결정일: 2026-03-19)

### PPT 서식/테마 설정 방식 (C - 혼합)
- 기본 내장 테마를 목적별로 내장
- 사용자가 `.pptx` 템플릿 파일 제공 시 해당 서식 우선 적용
- **테마 선택 전 샘플 이미지 미리보기 제공 필수** — 사용자가 시각적으로 확인 후 선택
- (결정일: 2026-03-19)

### 테마 미리보기 이미지 생성 방식
- **주방식 (B)**: python-pptx로 샘플 슬라이드 생성 → LibreOffice/Pillow로 이미지 변환 → Claude가 표시. 테마 수정 시 자동 반영
- **보조 활용 (C)**: AI 이미지 생성(BananaX 프롬프트 활용)은 인포그래픽 슬라이드 디자인 단계에서 적용
- (결정일: 2026-03-19)
